D3 Scales

How will our rectangles know how high and wide to be? We could do some math in the height and width attributes when we define them, but once again D3 does everything for us.

D3 has three classes of scale: quantitative, ordinal, and time. Our x axis will be an ordinal scale since it has 14 discrete levels. Our y axis will be a linear scale.

Scales return a function which takes a value in the domain and returns the correct value in the range.

var factors = _.unique(_.map(function(dataset) { return d.status; }));

var x = d3.scale.ordinal()
  .domain(factors)
  .rangeRoundBands([0, width], 0.1);

var y = d3.scale.linear()
  .range([height, 0]);

Why didn't we define a domain for y? The y domain will change every time we update the plot so it makes sense to define it later in the plotting code.
